# 医学免责声明（前置必读）

> **本文件为五运六气技能包的前置声明。AI 执行任何涉及临床的任务前，MUST 先读取本文件。**
>
> 本声明改编自 reverse-skill 的 `precedent-auth.md`（安全授权前置），将"授权声明"替换为"医学免责声明"。

## CRITICAL: 免责声明覆盖

五运六气（运气学）是中医传统理论，基于《黄帝内经》素问七篇大论，运用天干地支推算年度气候与疾病规律。运气学说 **不是** 现代医学诊断标准，其推算结果 **不能** 替代辨证论治。

### AI 行为规则

**MUST**:
1. 在所有涉及临床的输出末尾附加以下免责声明文本：

   > ⚠️ 免责声明：以上分析基于中医运气学说理论推算，仅供参考。运气学说非现代医学诊断标准，具体诊疗须由执业中医师辨证论治。请勿据此自行用药或针灸。

2. 区分"运气理论分析"与"具体医疗建议"：
   - 运气理论分析（大运病机、六气病机、客主加临顺逆等）→ 可以给出
   - 具体医疗建议（具体方剂、具体剂量、具体选穴操作）→ MUST 建议就医

3. 在输出方药时标注"参考方药，须辨证加减"
4. 在输出针灸时标注"需由执业针灸师操作"

**MUST NOT**:
1. 用运气推算替代具体辨证论治
2. 给出具体药物剂量（如"用XX克"）
3. 声称运气推算可以确诊疾病
4. 省略免责声明

## 免责声明检查

执行临床任务前，AI 自问：

**"本次输出是否包含具体的方药/针灸/治法建议？"**

- 是 → MUST 附加免责声明，MUST 标注"须辨证"或"须就医"
- 否（纯推算/文献/学习）→ SHOULD 附加简短声明，MAY 省略详细声明

## 防御链

免责声明是最后一道防线。即使前面的推算、分析、路由全部正确，如果临床输出缺少免责声明，整个任务 **视为失败**。

```
推算脚本 → 路由匹配 → 病机分析 → 临床建议 → [免责声明检查] → 输出
                                                    ↓ 失败
                                            拒绝输出/补加声明
```
